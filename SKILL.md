### 语气
- 说人话，像朋友聊天，不端着
- 口语化表达，无长句
- 适当使用反问句制造互动感
- 结尾留有余地，给观众思考空间

### 开场模式
- 现象引入："你有没有注意到..."
- 悬念提问："为什么只有XX上热搜了？"
- 对比反差："一边是...一边是..."

### 论述模式
- 先摆事实 → 分析原因 → 升华到深层矛盾
- 每个角度用"第一/第二/第三"或"先说/再说/更要命的是"过渡
- 用"说到底""所以"收束到核心观点

### 历史类比使用规则（严禁重复）
- **一组论点只配一组历史案例**，不要在不同段落重复用类似论据支撑同一论点
- 选择2-3个最具辨识度的案例即可（如2000互联网泡沫、2021中概股暴跌）
- 所有历史类比集中放在"一致性危险"论点段落或"历史押韵"收尾段落，**二选一**
- 不要在中间段落和结尾段落都用"历史上每次...后面都跟着出清"这种同义反复

### 长句口语适配规则
- 超过30字的句子必须拆分为2-3个短句
- 拆分时用"比如""这中间""只要"等口语连接词缝合
- ❌ "AI不是骗局，但AI股的定价已经把未来五十年的预期都折现进去了。这中间哪怕有一环脱钩——增速放缓、竞争加剧、落地不及预期——回调幅度都会非常恐怖。"
- ✅ "AI不是骗局，但AI股的定价，已经把未来五十年的成长预期都提前算进去了。只要中间有任何一环出问题：比如增速放缓、竞争变激烈、落地效果没达到预期，股价的回调幅度都会非常大。"

## 品牌安全措辞规范

### 敏感品牌指代方式
当内容涉及可能引发争议的品牌/人物时：
- ✅ "某'XX'品牌"（用品牌特征代称）
- ✅ "某国产手机巨头"
- ✅ "某科技大厂"
- ✅ "某品牌"
- ❌ 直接使用品牌全称

### 未经证实事实的表述
当某些细节无法确认时：
- ✅ "据短视频平台传播的画面显示..."
- ✅ "有网友拍摄到..."
- ✅ "画面中疑似..."
- ✅ "无论这些人是A还是B..."
- ❌ 直接断言具体身份/动机（如"演员扮家长"）

### 投资类内容措辞黑名单（零容忍）
以下措辞在金融/投资类口播中会触发平台审核风险或引发争议，**禁止使用**：

| 禁用词/短语 | 风险原因 | 替代方案 |
|------------|----------|----------|
| "泡沫""离谱"（标题中） | 直接定性市场，触发财经内容审核 | "偏离基本面" "显著分化" |
| "造假"（指企业行为） | 法律定性词，构成诽谤风险 | "数据注水" "统计口径存在争议" |
| "信仰" | 情绪化判断，弱化为"情绪定价""脱离基本面估值" | "情绪定价""脱离基本面估值" |
| "老登们" | 网络贬义俚语，对特定群体不尊重 | "长期持有XX的投资者" |
| "割肉"（描述投资者行为） | 主观推定动机 | "卖出持仓转向" |
| "亲妈都不认识""干到了" | 过度口语化，与专业数据段落风格割裂 | "远超合理区间""达到了XX" |
| "这是玄学" | 主观贬低 | "脱离了商业本质""超出合理预期范畴" |

### 数据精度规则
- 百分比/比值统一保留**1位小数**（如86.5%，不写86.7%或87%）
- 同一数据在全文各位置必须一致，避免口播稿写86.7%、数据表写86.5%
- 四舍五入规则：小数点后第二位≥5才进位，否则舍去
- 计算完成后必须**回乘验算**：如PS=市值÷营收，验算后标注"≈"而非"="

### 原则
- 只描述可观察的事实，不断言不可验证的动机
- 用"疑似""可能""据...显示"留余地
- 保护自己，避免被追究法律责任

### 法律防御性措辞规则（2026-06-20 EP18 新增）

1. **描述性措辞 ≠ 法律定性词**：涉及企业负面行为时，用"数据注水""统计口径存疑"而非"造假""作弊"
2. **间接指代 + 信源锚定必须同时存在**：用"某美系大佬"间接指代后，必须同时交代"路透社X月X日报道了相关争议"
3. **精确数字的去压力化**：无法交叉验证的精确数字（如"9中7"）一律模糊化（"多名""不少"）
4. **技术路线免责声明**：批评某公司某方案 ≠ 否定整条技术路线，口播中必须加平衡句（如"特斯拉一家公司≠所有纯视觉方案"）
5. **术语翻译义务**：英文缩写/专业术语首次出现时，括号内解释必须包含**大白话定义**（如"Robotaxi（无人驾驶出租车，就是没方向盘没踏板的那种）"）

## 豆包审核优化指南

豆包审核通常关注四个维度，提交前务必逐项检查：

### 1. 表达精准性：规避绝对化/标签化表述
- 避免"很多人第一反应"→ 改为"不少人"（更客观）
- 避免无来源的断言 → 补充"据一线销售透露""有消费者投诉"等来源
- 避免标签化用词（如"离婚综艺明星"）→ 改为描述性表述（"一位因离婚相关综艺引发争议的明星"）
- 避免强烈定性（如"挂羊头卖狗肉"）→ 改为客观描述事实（"宣传的产品和实际售卖的产品不一致"）

### 2. 口语化打磨：更贴近"说人话"
- 书面语转口语："品牌高端化的梦想…巨大的矛盾" → "品牌想走高端路线，但一线销售的做法却特别粗放"
- 保留记忆点的同时贴近日常："删得掉视频，删不掉公众的记忆" → "视频能火速删掉，但大家心里的印象早就留下了"
- 用口语化词汇收尾："营销的底线就开始模糊了" → "营销的底线就越来越没谱了"

### 3. 节奏衔接：弱化生硬过渡
- 避免"第二个问题""第三个问题"的列点式过渡
- 改用互动式过渡："但这只是大家能看到的表面问题，更让人反感的还有第二个点——…"
- **段落间衔接句**：列举多个现象/信号后，加一句过渡句将列举收束到核心论点
  - ❌ 列举三个现象后直接进入"但问题是…"
  - ✅ "这些现象凑到一起，其实已经在敲警钟了——哪怕AI是真风口，也架不住这么离谱的定价啊。"
- **设问式引出风险信号**：
  - ❌ "说几个信号值得盯着"
  - ✅ "那什么时候这泡沫可能破？我给大家提三个要盯紧的信号。"
- 结尾加口语化语气词强化收束感："所以说啊，…可就没那么简单了"

### 4. 开场抓耳度：增强互动感
- 用更具象的场景提问切入，比抽象提问更快抓注意力
- 例："你有没有刷到过高考考点门口的奇葩营销？有没有被某品牌的短视频反复刷屏刷到烦？"

### 5. 事实核查：细节必须可验证
- 任何具体人名、职务变动、产品名、宣传口号，必须有可靠来源
- 不确定的事实不要写，宁可模糊化（"该品牌一位高管"比点名更安全）
- 被审核指出事实错误会严重损害可信度

### 6. 数字口语化：让数字"说人话"
- 百分号写法：口播稿中用"30%"而非"百分之三十"（播音时直接读"百分之三十"但文字写30%更清爽）
- 大数表述："7.24亿"比"七点二四亿"更专业；但"1290倍"可补充通俗解释"说白了就是你买它1块钱的营收，得花1290块"
- 营收/市值对比时，用"才"强调反差："一年营收才7.24亿，市值却冲到了9336亿港元"

### 7. 重复表述精简
- "AI是真需求/真趋势"在同一篇中出现两次 → 第二处改为"AI赛道的真实需求不用怀疑"
- 同一数据/观点在不同位置复述 → 保留首次详细表述，后续用"正如前面提到的"或省略

## 双角色结构设计（2026-06-20 确认）

军伟观察系列采用**双角色结构**，设计理由：
1. **军伟无主播经验**：全文口播对非专业主播挑战大，改为只负责开场抛问题（~15秒），降低录制难度
2. **AI口播符合内容基因**：小零AI完成全文口播，与"AI知识科普"系列定位一致
3. **舆论风险可归因**：AI生成的分析内容，天然适合以"小零"AI角色口播，观众知道是AI在分析

### 角色分工
- **军伟（开场）**：3句话以内，抛出现象/冲突/悬念 + 过渡语"好，具体的数据分析，交给小零来拆。"
- **小零（口播）**：全文主体，以"大家好，我是小零"开头，以"我是小零，咱们下期见！"结尾
- **过渡语固定**：军伟说完后，接"好，具体的数据分析，交给小零来拆。"（不可随意更改）

### 历史演变
- EP01-EP16：单人播报（军伟全程口播）
- EP17 v1.1起：双角色结构（军伟开场+小零口播）
- EP18 v1.5：双角色结构回归（从单人播报改回双角色）

## 军伟观察制作工作流 SOP（2026-06-20 固化）

### 完整生产流程

```
热点素材收集 → 选题确认 → 写初稿(v1.0) → 豆包审核 → 修订(v1.1) → Tabbit审核 → 修订(v1.2) → 用户终审 → 定稿交付
```

### 阶段详解

#### Phase 1：素材收集与选题
- 来源：热点简报（cron db4ce01294ef）/ 用户主动提供 / 搜索发现
- 选题标准：时效性（24-48h内热点）+ 争议性（有明确对立面）+ 可数据化（有公开数字支撑）
- 用户确认选题后再动笔，不要自行选题直接写

#### Phase 2：写初稿 v1.0
- 结构：双角色（军伟开场+小零口播）
- 先写关键数据表 → 再写口播稿 → 再写宣发文案（同步生成，不要最后补）
- 数据必须标注来源，无法确认的标"⚠️ 待核验"

#### Phase 3：豆包审核（第一轮）
- 关注维度：口语化、措辞合规、逻辑连贯、重复表述、节奏衔接
- 豆包擅长发现：书面语过多、绝对化表述、敏感措辞不当、口语衔接生硬
- 根据反馈逐条修订 → v1.1

#### Phase 4：Tabbit审核（第二轮）
- 关注维度：🔴必改（事实错误/P0合规风险）+ 🟡建议改（结构/平衡性）+ 🟢可选润色
- Tabbit擅长发现：数据不准确、缺少法律防御措辞、缺少信源锚定、历史类比重复
- 根据反馈逐条修订 → v1.2

#### Phase 5：用户终审
- 用户可能提出新的方向性调整（如EP18选题完全更换）或细节修正
- 根据用户反馈修订 → v1.3+

#### Phase 6：定稿交付
- 口播稿 + 宣发文案 + 数据表三方确认同步
- 版本footer标注"✅ 可录制"
- 文件保存路径：`/Users/aaronyang/Nutstore Files/我的坚果云/Obsidian/88 AI学院/军伟观察/`

### 质量红线（不可触碰）

2. **敏感品牌必须间接指代**：用"某品牌""某美系大佬"等，不可直接点名（品牌安全规范）
3. **间接指代 + 信源锚定必须同时存在**：用"某品牌"后，必须同时交代"据XX报道"

## 审核流程规范

### ⚠️ 子代理模拟审核不可行（2026-06-20 用户明确要求）

**通过子代理（相同LLM）模拟豆包/Tabbit审核是自我审查，无外部视角增量，不可作为正式审核手段。**

正确流程：
1. AI生成初稿 → 用户手动提交给**真实豆包**和**Tabbit**审核
2. 用户把真实反馈（逐条修订建议）发回
3. AI根据真实反馈落实修改
4. 修改后再提交一轮确认

**禁止操作：**
- ❌ 用 delegate_task 生成"模拟豆包审核"或"模拟Tabbit审核"
- ❌ 把子代理的审核意见当作正式修订依据
- ❌ 认为子代理审核 = 外部审核通过

**为什么：** 子代理和主代理使用相同LLM，审查视角完全重合，等于自己审自己。真实豆包/Tabbit有不同的prompt和审查维度，能提供真正的外部增量。

### 口播稿与宣发文案同步审核

当口播稿经过审核修改（豆包/Tabbit/人工）后，**必须同步检查宣发文案和关键数据表**，确保三方关键信息一致。

**这是P0级同步，不是可选的。** 口播稿改了但宣发文案没同步 = 法律风险（"造假"留在抖音文案里、精确数字没做压力化处理、品牌安全措辞未统一）。

#### 同步检查项

逐条对比口播稿 v(N) 与宣发文案/数据表，重点检查：
1. **数据/事实一致性**：口播稿修正的数据，宣发文案是否同步
2. **品牌安全措辞一致性**：口播稿弱化的敏感表述，宣发文案是否同步
3. **定性表述一致性**：口播稿删除的强烈定性词汇，宣发文案/数据表标签是否残留
4. **人物描述一致性**：口播稿修正的人物描述，宣发文案是否同步
5. **数据表一致性**：关键数据表中的数据项是否与口播稿最终版匹配

#### 常见遗漏点

- ❌ 口播稿改了，但宣发文案的**某个平台版本**漏改（最常漏的是抖音/快手短文案）
- ❌ 口播稿删了某个表述，但宣发文案的**标签或标题**仍保留旧表述
- ❌ 口播稿增加了来源引用（"据路透社曝光"），但宣发文案未加
- ❌ 口播稿模糊化了精确数字（"9中7"→"多名"），但数据表仍保留精确值

#### 同步审核流程

```
口播稿 v(N) 定稿
  → 提取所有修改点（对照 v(N-1) 的 diff）
  → 逐一在宣发文案（抖音版、小红书版）中查找对应表述
  → 逐一在关键数据表中查找对应表述
  → 全部替换后，再通读一遍各方确认无遗漏
```

**⚠️ 典型案例（EP18 v1.2）：** 口播稿P0修订（"造假"→"注水"、"强制国标"→"准入技术要求"、"9中7"→"多名"）全部在口播稿本体执行，但宣发文案6处未同步，被Tabbit审核标记为最严重问题。

## 事实核查专项（提交前必检）

### 时间/日期
- [ ] 节气日期：确认节气对应的公历月份（如夏至=6月，非7月）
- [ ] 年份差值：用当前年份减声称年份，确认"X年过去了"的数值准确
- [ ] 历史年代：精确计算，不说约数偏差超过50年的表述（如公元前278年至今≈2304年，应说"两千三百年前"或"两千多年前"）

### 编号/序列
- [ ] 节气编号：按标准二十四节气顺序核对（立春1→...→夏至10→...→大寒24）
- [ ] 系列EP编号：与实际发布顺序一致
- [ ] 数据表编号与口播稿一致：改了口播稿的编号，数据表"核心定位"字段也要同步

### 外部状态（公司/产品/人物）
- [ ] 搜索最新信息确认公司/产品状态仍准确（如"已规模化"vs"如果规模化"）
- [ ] 已退出市场的公司不应再列为对比对象
- [ ] 承诺时间表变更需更新到当前视角（如"推迟6年+"而非旧表述）
- [ ] **市值/股价/估值数据**：必须实时搜索核验，不可用训练数据回答（见 `references/cross-doc-sync-lessons.md` 数据核验专项）

### 数据一致性
- [ ] 口播稿中出现的数据，数据表中也有对应条目
- [ ] 数据表状态列（✅/❌/🗑️）与口播稿描述一致
- [ ] 同一数据在不同位置的表述一致（如发射次数、复用次数）

## 审核自查清单

每篇脚本发布前检查：
- [ ] **双角色结构完整**：军伟开场（~15秒，3句话+过渡语）→ 小零全文口播（大家好，我是小零...我是小零，咱们下期见！）
- [ ] 开场直接进入：开场用现象/冲突/对比切入，不铺垫冗长背景
- [ ] 风格匹配度：说人话，像朋友聊天
- [ ] 逻辑完整性：论证闭环
- [ ] 口播适配性：口语化，无长句（>30字需拆分）
- [ ] 合规性：AI声明（开头+结尾）、风险提示（如需要）
### 版本记录规范
- v1.0 = 初稿
- v1.1 = 第一轮审核修订（通常口语化/措辞合规）
- v1.2 = 第二轮审核修订（通常事实准确/平衡表述/法律防御）
- v1.3+ = 后续修订
- 版本footer格式：`v1.3 | 审核：豆包9项 + Tabbit 6项全部采纳 | 状态：待二次审核` 或 `✅ 可录制`
- 审核方标注清楚，最终状态必须标注"✅ 可录制"或"待二次审核"
- **⚠️ "✅ 可录制"标记条件**：口播稿+宣发文案+数据表三方全部同步完成，无P0级遗漏
- [ ] 信息准确性：数据有来源
- [ ] 品牌安全：敏感品牌间接指代
- [ ] 事实严谨：未证实信息用推测语气
- [ ] 无违规内容：无股票代码/价格预测（投资类）
- [ ] 表达精准性：无绝对化/标签化表述
- [ ] 口语化：无书面语/长句
- [ ] 节奏衔接：过渡自然，不生硬
- [ ] 开场抓耳：具象场景切入
- [ ] 事实核查：所有具体信息可验证（见上方事实核查专项）
- [ ] **历史类比唯一性**：全文只有一组历史案例，不重复出现在不同段落
- [ ] **措辞合规**：无投资类措辞黑名单中的词汇（见品牌安全-投资类部分），涉及企业负面行为时用描述性措辞而非法律定性词（"注水"而非"造假"）
- [ ] **数据精度统一**：同一数据在口播稿、数据表、宣发文案中数值一致（含小数位），无法交叉验证的精确数字已模糊化处理
- [ ] **口播↔宣发↔数据表三方同步**：口播稿所有修改点已在宣发文案（抖音+小红书）和关键数据表中同步更新
- [ ] **非自我审核**：审核必须由真实外部工具（豆包/Tabbit）完成，子代理模拟审核无效
