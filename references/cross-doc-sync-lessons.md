# 口播稿↔宣发文案 同步审核教训

> 来源：2026-06-11 某"遥遥领先"品牌全家桶营销翻车 + 2026-06-17 SpaceX故事大王人设分析
> 场景：口播稿经豆包+Tabbit两轮审核修改至v1.1，但宣发文案/数据表未同步更新

## 案例一：品牌营销翻车（2026-06-11）

口播稿 v1.2 修改了以下关键表述，但宣发文案仍保留旧版：

| 修改项 | 口播稿 v1.2 | 宣发文案（旧） | 平台 |
|--------|-------------|---------------|------|
| 经销商描述 | "从南方到北方，大量经销商" | "全国经销商" | 抖音/快手、小红书、B站 |
| 交警表述 | "明确表态'不允许'" | "表态'不允许，已介入调查'" | 小红书、B站 |
| 代言描述 | "因个人生活话题引发广泛讨论的艺人" | "争议明星代言" | 小红书、B站 |
| 门店描述 | 客观描述KPI压力 | "挂羊头卖狗肉" | 抖音/快手 |

## 案例二：SpaceX 人设分析（2026-06-17）

口播稿 v1.1 做了18项修改，但以下文档部分未同步：

| 修改项 | 口播稿 v1.1 | 未同步位置 | 严重度 |
|--------|-------------|-----------|--------|
| 节气编号"第九首"→"第十首" | 口播稿全部位置 | 数据表"核心定位"字段仍写"第九首" | 🔴 |
| 时间表述"七月"→"六月" | 口播稿全部位置 | 封面提示词中未涉及但数据表未更新 | 🟡 |
| Waymo状态更新 | "已规模化运营" | 数据表旧版未记录 | 🟡 |

### 关键发现

数据表（## 关键数据）是独立于口播稿和宣发文案的第三个需要同步区域。

## 根因分析

1. 修改口播稿时，没有同步检查宣发文案和数据表
2. 宣发文案有4个平台版本（抖音/快手、小红书、B站、封面提示词），容易遗漏某个平台
3. **数据表是常被遗忘的第三个区域**：编号、时间、状态等变更需要同步到数据表
4. 短文案（抖音/快手≤100字）最容易漏改，因为字数限制导致表述更紧凑

## 教训

- **改口播稿必查宣发文案+数据表**：每次口播稿修改后，必须逐平台检查宣发文案，并同步数据表
- **建立对照表**：修改口播稿时，列出所有修改点，逐一在宣发文案和数据表中定位
- **短文案是重灾区**：字数受限的文案最容易残留旧表述，需重点检查
- **数据表独立检查**：编号、时间、状态变更不要以为"口播稿改了就行"，数据表是独立展示区域

## 完整同步检查流程

```
口播稿 v(N) 定稿
  → 提取所有修改点（对照 v(N-1) 的 diff）
  → 逐一在【宣发文案】中查找对应表述 → 替换
  → 逐一在【关键数据表】中查找对应信息 → 替换
  → 检查【封面提示词】中是否有相关表述 → 替换
  → 全部替换后，通读全部文档确认无遗漏
```

## 事实核查教训（2026-06-17 新增）

### 时间/日期事实
- 夏至节气在六月（公历6月21日前后），不是七月
- 2016年马斯克承诺"2024年送人上火星"，到2026年是10年，不是"10年过去了"表述需精确
- 屈原投江公元前278年，距今约2304年，"两千年前"偏差300年

### 编号/序列事实
- 夏至是二十四节气第十个（非第九个），系列编号需按实际节气顺序核对
- 口播稿编号改了，数据表定位也要同步

### 外部状态事实
- Waymo 已在 Phoenix/SF/LA 多城商业化运营（非"如果规模化"）
- Cruise 已基本退出市场（不应再列为对比对象）

### 核查方法
- 涉及日期：用当前年份减去声称年份，确认差值
- 涉及节气顺序：立春(1)→...→夏至(10)，逐个数一遍
- 涉及公司状态：搜索最新信息确认是否仍准确
- 涉及历史年代：精确计算，不说约数偏差超过50年的表述
- **涉及市值/股价/估值**：必须多源核验（web_search + 浏览器访问行情网站），不可凭记忆回答具体数字；首次回答时标注"需核验"或直接执行核验

## 数据核验专项（2026-06-20 新增）

### 场景
当文案/脚本中提及具体公司市值、股价、估值对比时，用户会追问核实具体数据。

### 核心原则
**不可凭训练数据给出精确数字。必须实时搜索核验，输出时标注数据来源和时点。**

### 核验流程
```
用户提及公司市值/股价/估值
  → web_search(公司名 最新市值 股价, freshness=pw)
  → 浏览器访问行情网站获取精确数据（Yahoo Finance/东方财富/华盛通）
  → 至少2个来源交叉比对
  → 输出时标注数据来源和时点
```

### 推荐数据源
| 市场 | 网站 | 特点 |
|------|------|------|
| 港股 | hk.finance.yahoo.com/quote/{code}.HK/key-statistics/ | 市值最精确 |
| 港股 | hstong.com/quotes/10000-{code}-HK | 实时股价 |
| A股 | quote.eastmoney.com/sz{code}.html | 实时行情 |
| A股 | push2.eastmoney.com/api/qt/stock/get?secid=0.{code}&fields=... | JSON API |

### 易错点
- 注意区分"市值"和"股价"，用户可能问的是PS/PE而非绝对股价
- 注意货币单位：A股=元人民币，港股=港元（市值比较时统一货币）
- 注意时点：非交易时间用最近交易日收盘数据
- 估值对比时计算倍数关系（如"X的PS是Y的N倍"）
