# 双角色结构转换教训

> 来源：2026-06-20 EP18 口播稿修订过程
> 触发：EP18 按单人播报写完后，用户明确要求切换为双角色结构

## 背景

EP18 最初按**单人播报（军伟全程口播）**编写至 v1.4，经豆包+Tabbit 双审核通过。但用户最终决定**切换回双角色结构**（军伟开场 + 小零口播），理由是：

1. 军伟无主播经验，全文口播挑战大
2. AI口播符合内容基因
3. 舆论风险可归因于AI

## 转换要点

### 单人播报 → 双角色 的具体改动

| 位置 | 单人播报 | 双角色 |
|------|----------|--------|
| 标题标签 | `## 口播稿（军伟单人播报）` | `## 口播稿` |
| 口播标注 | `**军伟（口播）：**` | `**军伟（开场）：**` + `**小零（口播）：**` |
| 开场 | 军伟直接进入正文分析 | 军伟3句话抛问题 + 过渡语 |
| 过渡 | 无 | "好，具体的数据分析，交给小零来拆。" |
| 开头 | （直接开始） | "大家好，我是小零。" |
| 结尾 | "我是军伟，咱们下期见！" | "我是小零，咱们下期见！" |
| 版本号 | v1.4（单人播报终稿） | v1.5（双角色终稿） |

### 军伟开场写作规则
- **3句话以内**：只抛现象/冲突/悬念，不展开分析
- **不出现数据**：具体数据交给小零
- **过渡语固定**："好，具体的数据分析，交给小零来拆。"
- **时长约15秒**（口播语速 ~4字/秒）

### 小零口播写作规则
- 以"大家好，我是小零。"开头（自我介绍，建立IP认知）
- 承接军伟开场的问题，展开完整分析
- 以"我是小零，咱们下期见！"结尾（固定收束语）
- 全文 900-1000字，约3-3:20口播时长

## 关键教训

**不要假设用户会接受"改进"方向**。单人播报看似更简洁，但对非专业主播来说实际录制难度更高。双角色是经过EP01-EP16实践验证过的结构，不应轻易放弃。

## 已确认的标准

EP17（SpaceX故事大王人设分析）和 EP18（智驾新规 vs 纯视觉路线之争）均为双角色结构定稿，后续 EP 应继续沿用。
